# Contributing

Thanks for taking the time to improve Modular Hill Generator.

## Before you open a pull request

- Keep changes focused on one problem or feature.
- Include the Unity version you tested with.
- For bugs, add reproduction steps and the expected versus actual result.
- For editor or workflow changes, attach screenshots or a short video.
- Avoid committing generated build artifacts unless they are part of the package itself.

## Good contribution targets

- Documentation fixes and setup clarifications.
- Better generation rules and inspector usability.
- Additional modular prefabs and visual variants.
- Sample scenes and demo content.
- Bug fixes in editor tooling or prefab export workflows.

## Style

- Prefer simple, maintainable changes over large abstractions.
- Keep runtime and editor responsibilities separate.
- Match the existing naming and folder structure when possible.
